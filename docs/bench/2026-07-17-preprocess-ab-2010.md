## 판정 (2026-07-17 truth-check로 정정됨 — 전처리는 유효하게 평가되지 않았음)

> ⚠️ 최초 결론("전처리 3종 효과 없음, 투자 불필요")은 적대적 재검증(truth-check + Codex 교차)에서 **실험 결함 3종**이 드러나 철회됐다. 아래는 정정된 판정이다.

- **넓은 결정 "전처리 투자 불필요"는 철회한다** — 전처리는 아래 결함 때문에 애초에 제대로 평가된 적이 없다:
  1. **지표 고장**: 현행 CER은 어순에 민감한 시퀀스 편집거리인데, 정답 라벨을 Bbox id순으로 join한다. 공공문서는 2차원 양식이라 id순 != OCR 읽기순서. 실제 페이지 대조 결과 OCR이 라벨보다 더 정확-완전하게 읽는데도 CER이 높게 나온다(라벨이 양식기호 ①②③, 기관명 헤더, 페이지번호를 누락). 순서 무관 글자 누락률로 보면 2010 세트 Vision의 진짜 놓침은 약 1%뿐 — 현행 CER 18%는 대부분 어순/라벨 결함.
  2. **JPEG 재압축 교란**: `apply_lever`가 각도 0(미변경)에서도 전처리본을 `.jpg`로 재저장(품질 재압축)한다. baseline은 원본을 그대로 쓴다. 따라서 전처리 3종 전부 baseline에 없는 재압축 손상을 안는다. 2010 deskew는 실제 회전이 2/50페이지뿐인데 출력은 28/50페이지에서 바뀌었다 — 나머지 26건은 순수 재압축 노이즈다.
  3. **deskew 미발동**: deskew는 2010-b1980 모두 50페이지 중 2페이지만 실제 회전. 스캔 문서가 거의 반듯해 deskew 자체가 사실상 테스트되지 않았다.
- **유지되는 좁은 결정**: (a) 이 고정 설정 3종을 모든 문서에 **무조건 기본 적용하지 않는다** (b) **upscale은 기각** — 처리시간 11.4배 비용은 재평가와 무관하게 강한 기각 근거.
- **deskew는 판정 보류** — 실제 기울어진 입력에서 재압축 없이 재측정하기 전까지 채택도 기각도 확정하지 않는다.
- 시사점: 현행 CER의 지배 요인은 전처리도 OCR 품질도 아닌 **라벨 정렬 방식(어순)과 라벨 불완전성**이다.

## 실험 결함 3종 (truth-check 2026-07-17, Codex 교차 검증)

1. **지표 고장 (가장 치명적)** — 현행 CER은 어순 민감 시퀀스 편집거리인데 정답이 Bbox id순 join이라, 2차원 양식 문서에서 OCR 읽기순서와 어긋난다. 순서 무관 글자 누락률로 근사한 Vision의 진짜 놓침: 2010 약 1.0%, 1990 4.4%, 1980 5.7%, b1980 17.3%. 현행 CER(18~53%)의 대부분은 어순-라벨불완전성이지 OCR 오류가 아니다. (Codex 주의: 순서무관 multiset은 위치 틀린 오인식을 놓쳐 과소평가하는 보조지표이며, CER은 공백 포함/누락률은 공백 제외라 정밀 비율 산식은 부정확 — 방향만 유효.)
2. **JPEG 재압축 교란** — `apply_lever`(preprocess.py:124-127)가 각도 0에서도 전처리본을 `.jpg` 재저장. baseline은 원본 사용. 전처리 3종 전부 baseline에 없는 재압축 손상 포함.
3. **deskew 미발동 + 표본 편향** — deskew 실제 회전 2/50(2010-b1980). 세트별 앞 50페이지라 무작위 아님. contrast는 2010만, upscale은 2010-b1980만, 고정 강도 1개씩만 시험.

참고: 1980 deskew는 순서무관 놓침률이 0.0573→0.0535로 줄어 "어느 품질대에서도 개선 없음"은 반증됨 (작고 불완전한 신호이나 deskew를 닫을 근거는 부족).

## 다음 단계 제안 (재평가 전제조건)

전처리 판정을 확정하려면 아래를 먼저 고쳐야 한다:
1. **재압축 제거**: deskew 각도 0(미변경)이면 전처리본 저장 없이 원본 경로를 그대로 반환하도록 수정 — 재압축 교란 제거.
2. **지표 교체**: bbox 좌표 기반 읽기순서 재정렬로 라벨을 복원하거나, 순서 무관 보조지표를 주지표로 병행 (스펙 개정 필요).
3. **결함 유형별 표본**: 실제 기울어진/저화질 페이지 30~50장을 사람이 완전 교정한 정답으로 재측정 + 페이지별 불확실성 범위 확인.
4. 위 재평가 전까지 파이프라인 기본값은 baseline 유지. upscale은 11.4배 비용으로 이미 기각.
5. 우선순위상 전처리 재평가보다 스펙 7절 LLM 보정 비교가 가성비 높음 — 단 같은 라벨 지표를 쓰지 않도록 위 2번을 선행해야 함.
