# 책 스캔 OCR 정식 구현 설계 (읽기 좋은 텍스트 변환기)

- 작성일: 2026-07-07
- 상태: 사용자 승인됨
- 선행: PoC 통과 (`docs/superpowers/specs/2026-07-06-korean-ocr-design.md`). PoC 코드 `ocr_book.py`는 폐기하고 전면 재작성한다.

## 1. 목표

책 스캔 폴더(jpg)를 입력받아 두 가지 산출물을 만드는 CLI 도구:

1. **페이지별 원본 txt** (검수용): OCR 줄 단위 그대로
2. **연속본 txt** (읽기용): 꼬리말 제거 + 문단 복원, 페이지 경계를 넘는 문단은 이어붙임

## 2. 확정된 결정사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 접근법 | 위치 좌표(bounding box) 기반 분석 | 꼬리말 오탈자("한탁해해 시작하기17")와 문단 과분할 문제를 위치로 회피. 사용자 A안 선택 |
| 출력 | 연속본 + 페이지별 원본 둘 다 | 검수와 읽기 모두 지원 |
| 재사용성 | CLI 도구 (입력/출력 경로 인자) | 다음 책(4장 이후)에 재사용 |
| 오탈자 교정 | 하지 않음 | 원문 충실. 자동 교정은 원문 왜곡 위험 |
| 코드 수준 | 정식: 타입 힌트 100%, 한국어 docstring, logging, 이름 있는 상수, TDD | CLAUDE.md 정식 구현 규칙 |

## 3. 실측 근거 (2026-07-07, PoC 산출물 31장 분석)

- 꼬리말 2종: 짝수 쪽 `24 주식시장을 이긴 전략들`, 홀수 쪽 `Chapter 01 전략매매 시작하기 23` (변형: `Chapter01`, 오탈자형 `한탁해해 시작하기17`, `주식시장을 이전 썬택는`)
- 31장 전부 마지막 줄이 꼬리말이었음 (헤더는 없음)
- 여러 페이지가 문장 중간에서 시작 → 페이지 경계 문단 병합이 필요
- 원본 이미지에서 새 문단 첫 줄은 들여쓰기됨 → x좌표로 감지 가능
- 본문 줄바꿈은 어절(단어) 경계에서 일어남 → 줄 연결 시 공백 1칸이 안전

## 4. 모듈 구성 (SRP)

```
img2txt/                파이썬 패키지
  __init__.py
  cli.py                명령 인자 해석(argparse), 전체 흐름 조립, 종료 코드
  scanner.py            jpg 수집 + 파일명 마지막 숫자 기준 자연 정렬
  ocr.py                이미지 1장 -> list[OcrLine]. ocrmac 래핑, EXIF 회전, y좌표 정렬
  layout.py             페이지 단위 분석: 꼬리말 식별, 제목 줄 분류, 문단 시작 감지
  assembler.py          페이지 목록 -> 문단 복원된 연속 텍스트 (경계 병합)
  writer.py             출력 파일 쓰기 (pages/*.txt + book.txt)
tests/
  test_scanner.py  test_layout.py  test_assembler.py  test_integration_ocr.py
```

핵심 데이터 모델:

- `OcrLine`: `text: str`, `confidence: float`, `x: float`, `y: float`, `width: float`, `height: float` (Vision 정규화 좌표, 좌하단 원점)
- `Page`: `number: int`, `lines: list[OcrLine]`
- layout/assembler는 OcrLine만 입력받는 순수 로직 → OCR 없이 합성 데이터로 단위 테스트

## 5. 핵심 규칙

1. **줄 순서**: 모든 줄을 y좌표 내림차순(위→아래)으로 정렬. Vision 반환 순서에 의존하지 않는다.
2. **꼬리말 제거**: y 중심이 페이지 최하단 띠(임계값 `FOOTER_BAND`) 안에 있는 줄을 모두 제거. 꼬리말을 하나도 못 찾은 페이지는 제거하지 않고 warning (과잉 삭제보다 잔존이 안전).
3. **문단 시작 감지**: 꼬리말과 제목 줄을 제외한 본문 줄들의 최소 x(왼쪽 여백)를 기준으로, `INDENT_MIN` 이상 오른쪽에서 시작하는 줄 = 문단 시작. (여백 추정에 제목/꼬리말이 섞이면 기준이 왜곡되므로 반드시 제외)
4. **제목 줄 예외**: 글자 높이가 본문 중앙값의 `TITLE_HEIGHT_RATIO`배 초과인 줄은 제목으로 분류, 병합하지 않고 독립 문단으로 유지 (챕터 시작 페이지의 "1", 장 제목 등).
5. **페이지 경계 병합**: 다음 페이지 첫 본문 줄이 문단 시작이 아니고 제목도 아니면, 이전 페이지 마지막 문단에 공백 1칸으로 이어붙인다.
6. **문단 내 줄 연결**: 공백 1칸. 연속본에서 문단 사이는 빈 줄 1개.
7. **임계값 캘리브레이션**: 구현 첫 단계에서 실제 3~5페이지(일반 본문, 챕터 시작, 곡면 왜곡)의 좌표를 덤프해 `FOOTER_BAND`, `INDENT_MIN`, `TITLE_HEIGHT_RATIO` 값을 결정하고, 근거 주석과 함께 이름 있는 상수로 기록한다.

## 6. CLI 사양

```
python -m img2txt <입력폴더> [-o <출력폴더>] [-v]
```

- `<입력폴더>`: jpg가 있는 폴더 (필수)
- `-o`: 출력 폴더, 기본 `./output`. `pages/` 하위와 `book.txt` 생성
- `-v`: DEBUG 로그
- 진행/요약은 logging (INFO): 페이지 진행, 마지막에 성공/실패/빈 결과/제거 꼬리말 수
- 종료 코드: 정상 0 (부분 실패 포함, warning으로 표시), 전체 실패 또는 입력 폴더에 jpg 없음 1

## 7. 에러 처리

- 이미지 1장 실패: warning + 다음 이미지 계속
- 빈 OCR 결과: warning, 원본 txt는 빈 파일로 기록, 연속본에서 제외
- 꼬리말 미감지 페이지: warning + 제거 생략
- 파일명에 숫자 없음: warning + 맨 뒤 배치

## 8. 테스트 전략 (TDD, 핵심 → 경계 → 에러 → 통합)

- `test_scanner`: 자연 정렬(2 < 10), 숫자 없는 파일, 빈 폴더
- `test_layout`: 꼬리말 감지(정상/오탈자형/미감지), 문단 시작 감지(들여쓰기/비들여쓰기), 제목 줄 분류
- `test_assembler`: 페이지 경계 이어짐/새 문단/제목 페이지/빈 페이지
- `test_integration_ocr`: 실제 이미지 1장 OCR (macOS 전용 marker, CI 제외 가능)

## 9. 최종 검증 기준

1. 전체 테스트 통과 (`pytest` 출력 첨부)
2. 실제 31장 실행: 연속본에서 꼬리말 패턴(`주식시장을 이긴 전략들` 단독 줄, `Chapter` 시작 줄) 잔존 0건
3. 원본 대조 3곳: 챕터 시작 페이지 1곳, 페이지 경계 문단 연결부 2곳

## 10. 알려진 한계 / 리스크

- 본문 OCR 오탈자는 그대로 남는다 (교정 안 함 방침)
- 들여쓰기 없이 시작하는 특수 문단(인용 블록 등)은 이전 문단에 붙을 수 있다 → 최종 검증에서 발견 시 규칙 보강
- 임계값은 이 책 기준으로 캘리브레이션된다. 다른 책에서 판형이 다르면 상수 조정이 필요할 수 있다 (상수에 근거 주석 필수)
