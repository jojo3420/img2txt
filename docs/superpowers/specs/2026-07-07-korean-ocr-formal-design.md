# 책 스캔 OCR 정식 구현 설계 (읽기 좋은 텍스트 변환기)

- 작성일: 2026-07-07
- 상태: 사용자 승인됨 (2026-07-07 수정: LLM 보정 도구 분리안(C안) 반영, 듀얼 리뷰 권고 Must 3건+Should 8건 반영 — `docs/review/review-result-dual-20260707-193809.md`)
- 선행: PoC 통과 (`docs/superpowers/specs/2026-07-06-korean-ocr-design.md`). PoC 코드 `ocr_book.py`는 폐기하고 전면 재작성한다.

## 1. 목표

독립 실행 가능한 두 개의 CLI 도구(서브커맨드)로 구성한다:

**변환 도구 (`convert`)** — 책 스캔 폴더(jpg)를 입력받아:

1. **페이지별 원본 txt** (검수용): OCR 줄 단위 그대로
2. **연속본 txt** (읽기용): 꼬리말 제거 + 문단 복원, 페이지 경계를 넘는 문단은 이어붙임

**보정 도구 (`correct`)** — 연속본 txt를 입력받아 로컬 LLM(Ollama)으로 OCR 오류를 보정:

3. **보정본 txt**: 오탈자("경단로"→"결단코")와 병합 띄어쓰기 오류("20 세기"→"20세기") 수정
4. **보정 기록 로그**: 바뀐 문단의 전/후 대조 (사람 검수용)

무료/유료 상품 구조와 대응: 무료 = 변환 도구만, 유료 = 변환 + 보정.

## 3. 실측 근거 (2026-07-07, PoC 산출물 31장 분석)

- 꼬리말 2종: 짝수 쪽 `24 주식시장을 이긴 전략들`, 홀수 쪽 `Chapter 01 전략매매 시작하기 23` (변형: `Chapter01`, 오탈자형 `한탁해해 시작하기17`, `주식시장을 이전 썬택는`)
- 31장 전부 마지막 줄이 꼬리말이었음 (헤더는 없음)
- 여러 페이지가 문장 중간에서 시작 → 페이지 경계 문단 병합이 필요
- 원본 이미지에서 새 문단 첫 줄은 들여쓰기됨 → x좌표로 감지 가능
- 본문 줄바꿈은 대체로 어절(단어) 경계에서 일어남 → 줄 연결 시 공백 1칸이 기본. 단, 단어 중간 줄바꿈 사례 실존("...되었다는 20 / 세기 최고의...") → 공백 연결 시 "20 세기" 같은 띄어쓰기 오류가 남으며, 이는 LLM 보정 도구가 다루는 유형
- 로컬 Ollama에 `gemma4:latest`(9.6GB) 설치 확인됨 (`ollama list`, 2026-07-07). 대안 후보: qwen3:14b, exaone3.5:7.8b

## 4. 모듈 구성 (SRP)

```
img2txt/                파이썬 패키지
  __init__.py
  cli.py                명령 인자 해석(argparse), 전체 흐름 조립, 종료 코드
  scanner.py            jpg/jpeg 수집(확장자 대소문자 무시) + 파일명 마지막 숫자 기준 자연 정렬
  ocr.py                이미지 1장 -> list[OcrLine]. ocrmac 래핑(지연 임포트: correct만 쓰는 환경에서 ocrmac 부재로 죽지 않게), EXIF 회전, y좌표 정렬
  layout.py             페이지 단위 분석: 꼬리말 식별, 제목 줄 분류, 문단 시작 감지
  assembler.py          페이지 목록 -> 문단 복원된 연속 텍스트 (경계 병합)
  corrector.py          Ollama HTTP 클라이언트 + 문단 단위 LLM 보정 + 안전장치
  writer.py             출력 파일 쓰기 (pages/*.txt + book.txt + book_corrected.txt + corrections.log)
tests/
  test_scanner.py  test_layout.py  test_assembler.py  test_corrector.py  test_integration_ocr.py
```

핵심 데이터 모델:

- `OcrLine`: `text: str`, `confidence: float`, `x: float`, `y: float`, `width: float`, `height: float` (Vision 정규화 좌표, 좌하단 원점)
- `Page`: `number: int`, `lines: list[OcrLine]`
- layout/assembler는 OcrLine만 입력받는 순수 로직 → OCR 없이 합성 데이터로 단위 테스트

## 5. 핵심 규칙

1. **줄 순서**: 모든 줄을 y좌표 내림차순(위→아래)으로 정렬. Vision 반환 순서에 의존하지 않는다.
2. **꼬리말 제거**: y 중심이 페이지 최하단 띠(임계값 `FOOTER_BAND`) 안에 있으면서, 보조 조건 — 줄에 숫자 포함 또는 본문 폭 대비 짧은 줄(`FOOTER_MAX_WIDTH_RATIO`) — 을 함께 만족하는 줄만 꼬리말로 제거. (위치만으로 전부 지우면 띠에 걸친 본문 줄이 오삭제될 수 있고, "미감지 시 생략" 안전장치도 위치 전용 정의에서는 작동하지 않는다.) 꼬리말 후보가 하나도 없는 페이지는 제거하지 않고 warning. 보조 조건 탓에 꼬리말이 남는 실패는 허용하되(과잉 삭제보다 잔존이 안전) 본문 오삭제는 불허.
3. **문단 시작 감지**: 꼬리말과 제목 줄을 제외한 본문 줄들의 최소 x(왼쪽 여백)를 기준으로, `INDENT_MIN` 이상 오른쪽에서 시작하는 줄 = 문단 시작. (여백 추정에 제목/꼬리말이 섞이면 기준이 왜곡되므로 반드시 제외)
4. **제목 줄 예외**: 글자 높이가 본문 중앙값의 `TITLE_HEIGHT_RATIO`배 초과인 줄은 제목으로 분류, 병합하지 않고 독립 문단으로 유지 (챕터 시작 페이지의 "1", 장 제목 등).
5. **페이지 경계 병합**: 다음 페이지 첫 본문 줄이 문단 시작이 아니고 제목도 아니면, 이전 페이지 마지막 문단에 공백 1칸으로 이어붙인다. 단, 직전 페이지가 OCR 실패 또는 빈 결과로 연속본에서 빠진 경우에는 병합하지 않는다 — 누락 페이지를 건너뛴 채 이어붙이면 문장이 소리 없이 훼손된다. 이때는 새 문단으로 시작하고 연속본에 `[페이지 N 누락]` 표식 + warning을 남긴다.
6. **문단 내 줄 연결**: 공백 1칸. 연속본에서 문단 사이는 빈 줄 1개.
7. **임계값 캘리브레이션**: 구현 첫 단계에서 실제 3~5페이지(일반 본문, 챕터 시작, 곡면 왜곡)의 좌표를 덤프해 `FOOTER_BAND`, `INDENT_MIN`, `TITLE_HEIGHT_RATIO` 값을 결정하고, 근거 주석과 함께 이름 있는 상수로 기록한다.

### LLM 보정 규칙 (correct 도구)

8. **문단 단위 보정**: 연속본을 문단(빈 줄 구분) 단위로 나눠 Ollama API(로컬 HTTP, 기본 `http://localhost:11434`)에 순차 요청한다.
9. **보정 범위 제약 프롬프트**: "OCR 오류(오탈자, 잘못된 띄어쓰기)만 수정하고 문장 재작성-내용 추가-삭제-요약 금지"를 시스템 지시로 고정한다. 프롬프트에 실측 오류 쌍 2~3개("경단로"→"결단코", "20 세기"→"20세기")를 few-shot 예시로 포함한다.
10. **길이 가드**: 보정 결과와 원문의 길이 차가 `max(MIN_DIFF_CHARS, 원문 길이 * LENGTH_GUARD_RATIO)`(초기값: 5자, 10%, 캘리브레이션으로 결정)를 넘으면 보정을 버리고 원문 유지 + warning. 비율만 쓰면 제목 같은 짧은 문단에서 정당한 보정까지 차단되므로 절대 하한(`MIN_DIFF_CHARS`)을 둔다. 또한 `MAX_PARA_CHARS`를 초과하는 비정상적으로 긴 문단(문단 감지 실패 의심)은 보정을 생략하고 원문 유지 + warning.
11. **실패 폴백**: Ollama 미응답-타임아웃-예외 시 해당 문단은 원문 유지하고 계속 진행. 보정은 향상 수단이지 단일 장애점이 아니다.
12. **보정 기록**: 보정 반영/가드 차단/실패가 발생한 문단마다 문단 번호, 상태(보정/가드 차단/실패), 사유, 모델명, 전/후 텍스트를 `corrections.log`에 남긴다. 원문 `book.txt`는 절대 수정하지 않는다.
13. **모델 품질 게이트**: 구현 중 샘플 3문단(오탈자 포함 문단, "20 세기" 병합 문단, 정상 문단)으로 gemma4 보정 품질을 확인한다. 정상 문단을 불필요하게 바꾸거나 오류를 못 잡으면 qwen3:14b, exaone3.5:7.8b와 비교해 기본 모델을 결정한다.

## 6. CLI 사양 (서브커맨드 2개, 독립 실행)

```
python -m img2txt convert <입력폴더> [-o <출력폴더>] [-v]
python -m img2txt correct <연속본txt> [--model gemma4:latest] [-o <출력폴더>] [-v]
```

**convert (변환 도구)**

- `<입력폴더>`: jpg/jpeg가 있는 폴더 (필수, 확장자 대소문자 무시)
- `-o`: 출력 폴더, 기본 `./output`. `pages/` 하위와 `book.txt` 생성
- 진행/요약은 logging (INFO): 페이지 진행, 마지막에 성공/실패/빈 결과/제거 꼬리말 수
- 종료 코드: 정상 0 (부분 실패 포함, warning으로 표시), 전체 실패 또는 입력 폴더에 jpg 없음 1

**correct (보정 도구)**

- `<연속본txt>`: convert가 만든 book.txt 등 텍스트 파일 (필수)
- `--model`: Ollama 모델명, 기본 `gemma4:latest`
- `-o`: 출력 폴더, 기본 입력 파일과 같은 폴더. `book_corrected.txt`와 `corrections.log` 생성
- 진행 로그: 문단 N/전체, 마지막에 보정/유지/가드 차단/실패 문단 수 요약
- 종료 코드: 정상 0 (문단 일부 실패 포함), Ollama 접속 불가 또는 모델 없음 1, 전체 문단의 보정 요청 실패(정상 응답 0건)도 1 (명확한 안내 메시지 출력). 실행 도중 Ollama가 죽어 전 문단이 원문 폴백된 결과물이 정상 종료로 보이는 Silent Failure를 막기 위함. 단순히 변경할 오류가 없어 전 문단이 원문 그대로인 경우(응답은 정상 수신)는 정상 0

공통: `-v`는 DEBUG 로그. 모든 파일 입출력은 `encoding='utf-8'` 고정. 기존 출력 파일이 있으면 덮어쓴다 (캘리브레이션-재실행이 기본 사용 흐름이므로 기본 차단은 두지 않는다).

## 7. 에러 처리

- 이미지 1장 실패: warning + 다음 이미지 계속
- 빈 OCR 결과: warning, 원본 txt는 빈 파일로 기록, 연속본에서 제외. 다음 페이지와의 경계 병합 금지 + `[페이지 N 누락]` 표식 (규칙 5)
- 꼬리말 미감지 페이지: warning + 제거 생략
- 파일명에 숫자 없음: warning + 맨 뒤 배치
- (correct) Ollama 접속 불가/모델 미설치: 시작 시 사전 점검 후 안내 메시지와 함께 종료 코드 1
- (correct) 문단 보정 실패/타임아웃/길이 가드 초과: 원문 유지 + warning, 계속 진행
- (correct) 전체 문단 보정 요청 실패(정상 응답 0건): 종료 코드 1 (Silent Failure 방지)

## 8. 테스트 전략 (TDD, 핵심 → 경계 → 에러 → 통합)

- `test_scanner`: 자연 정렬(2 < 10), 숫자 없는 파일, 빈 폴더
- `test_layout`: 꼬리말 감지(정상/오탈자형/미감지/띠 안이지만 보조 조건 불충족 시 잔존), 문단 시작 감지(들여쓰기/비들여쓰기), 제목 줄 분류
- `test_assembler`: 페이지 경계 이어짐/새 문단/제목 페이지/빈 페이지(병합 금지 + 누락 표식)
- `test_corrector`: Ollama HTTP는 모킹(가짜 응답). 길이 가드 차단(비율 + 절대 하한), 긴 문단 보정 생략, 실패 폴백(원문 유지), 전체 실패 시 종료 코드 1, 보정 기록 생성, 정상 보정 반영
- `test_integration_ocr`: 실제 이미지 1장 OCR (macOS 전용 marker, CI 제외 가능)

## 9. 최종 검증 기준

1. 전체 테스트 통과 (`pytest` 출력 첨부)
2. 실제 31장 convert 실행: 연속본에서 꼬리말 패턴(`주식시장을 이긴 전략들` 단독 줄, `Chapter` 시작 줄) 잔존 0건
3. 원본 대조 3곳: 챕터 시작 페이지 1곳, 페이지 경계 문단 연결부 2곳 (실측 사례 "지원해 / 주겠다는"이 한 문단으로 복원되는지 포함)
4. correct 실행: corrections.log의 변경 문단 표본 5개를 사람이 확인 — 실제 오류 수정("경단로", "20 세기" 유형)이면서 원문 의미 변형이 없어야 함

## 10. 알려진 한계 / 리스크
