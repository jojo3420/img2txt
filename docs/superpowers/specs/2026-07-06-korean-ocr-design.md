## 1. 목표

`/Users/joel.silver/Documents/주식시장을 이긴 전략들/1,2,3장` 폴더의 jpg 31장(책 촬영본)을

## 2. 확정된 결정사항

| 항목 | 결정 | 근거 |
|------|------|------|
| OCR 엔진 | Apple Vision (ocrmac 래퍼) 단독 | macOS 내장, 설치 최소(`pip install ocrmac`), 한글 인쇄물 인식률 상위. 사용자가 비교안(C안) 대신 A안 단독 선택 |
| 출력 형태 | 페이지별 txt + 합본 txt 둘 다 | 검수(페이지별)와 활용(합본) 모두 필요 |
| 작업 수준 | PoC: happy path만, 경로 하드코딩, 테스트/타입 힌트 생략. 단, 이미지 1장 실패로 전체가 중단되지 않도록 최소 실패 처리(실패 파일명 출력 후 계속)만 포함 | CLAUDE.md PoC 규칙 + 듀얼 리뷰 권고 |
| 언어 설정 | `language_preference=['ko-KR']` | 한국어 인식률 확보. 한자/혼합 레이아웃 문제 시 livetext 백엔드 시도 |

## 3. 구성

스크립트 1개: `ocr_book.py`

데이터 흐름:

1. 대상 폴더에서 `*.jpg` 수집
2. 자연 정렬: 파일명의 **마지막 숫자**를 추출해 숫자 크기순 정렬.
   숫자가 없는 파일은 경고 로그 후 맨 뒤에 배치한다.
   (사전순 정렬은 "- 10.jpg"가 "- 2.jpg"보다 앞서므로 페이지 순서가 깨진다.
   실측 확인: 현재 데이터 31장은 마지막 숫자가 2~32로 중복 없이 정렬된다)
3. 저장 전 출력 디렉터리를 생성한다 (`os.makedirs(..., exist_ok=True)`).
   출력 위치는 프로젝트 루트(img2txt) 기준 `output/`이다.
4. 각 이미지를 EXIF 회전 반영하여 열고 Apple Vision OCR 실행.
   1장이 실패하면 실패 파일명을 출력하고 다음 이미지로 계속 진행한다.
5. 저장:
   - `output/pages/<원본파일명(확장자 제외)>.txt` — 이미지 1장당 1개
   - `output/book.txt` — `===== p.<파일명 숫자> =====` 구분선으로 이어 붙인 합본
     (구분선 숫자는 파일명 숫자를 그대로 사용하므로 2~32가 된다)
6. 페이지마다 진행 로그 1줄 출력 (몇 번째/전체, 파일명)

## 4. PoC 검증 기준 (판정 1개)

Vision OCR은 처리 속도가 빨라 31장 전체를 한 번에 변환한 뒤,
검수 3장을 골라 원본 이미지와 눈으로 대조한다.
검수 3장은 미리 정한다: 첫 페이지, 일반 본문 페이지, 곡면 왜곡이 큰 페이지.

**판정: 아래 3개 조건을 모두 충족하는가?**

1. 31장 중 빈 OCR 결과가 0장
2. 검수 3장의 본문 문장이 흐름을 따라 판독 가능 (곡면 왜곡부 포함)
3. 줄 순서 뒤섞임 없음 (위에서 아래 순서 유지)

- 통과 → 결과 그대로 사용, 정식 구현 여부 결정
- 미흡 → 조건 3만 실패면 Y좌표 정렬 헬퍼를 먼저 시도,
  그 외에는 PaddleOCR(오픈소스) 비교 투입을 재검토

## 5. 고려사항과 알려진 한계

- **곡면 왜곡**: 책 접합부 쪽 글자가 휘어 있어 해당 부분 오인식 가능. PoC에서는 측정만 하고, 심하면 정식 단계에서 이미지 전처리를 추가한다.
- **텍스트 블록 순서**: Apple Vision은 인식된 텍스트 블록을 항상 위→아래 순서로 반환한다고 보장하지 않는다. 단일 컬럼 책 본문이라 위험은 낮지만 검수 조건 3에서 확인하고, 뒤섞이면 바운딩 박스 Y좌표(세로 위치) 기준 정렬 헬퍼를 추가한다.
- **줄바꿈**: OCR 결과는 시각적 줄 단위라 문장 중간 줄바꿈이 남는다. PoC에서는 그대로 두고, 정식 단계에서 "문장부호로 끝나지 않으면 다음 줄과 병합" 규칙으로 문단을 복원한다.
- **꼬리말**: 페이지 하단의 "24 주식시장을 이긴 전략들" 같은 페이지 번호+제목이 본문에 섞인다. PoC에서는 그대로 둔다.
- **저작권**: 소장 도서의 개인 이용 목적 변환은 사적 복제 범위. 추출 텍스트의 외부 공유는 별개 문제.

## 6. 정식 구현 시 확장 항목 (PoC 범위 밖)

- 에러 처리(깨진 이미지, 빈 결과), 진행률 표시, 경로를 CLI 인자로
- 문단 복원 후처리, 꼬리말 제거 규칙
- 표/숫자 검수: 투자 전략서라 수치가 핵심 정보이므로, 숫자와 표의 오인식 검증을 일반 문장보다 우선한다
- 확장자 변형(`.JPG`, `.jpeg`) 대응, natsort(자연 정렬 라이브러리) 도입 검토
- 타입 힌트 100%, logging, 테스트
