# 한글 책 스캔 OCR 변환기 설계 (PoC)

- 작성일: 2026-07-06
- 상태: 사용자 승인됨 (엔진은 A안 단독으로 확정)
- 작업 수준: PoC 먼저, 결과가 만족스러우면 정식 구현으로 확장

## 1. 목표

`/Users/joel.silver/Documents/주식시장을 이긴 전략들/1,2,3장` 폴더의 jpg 31장(책 촬영본)을
한국어 OCR로 텍스트 변환하여 페이지별 txt와 합본 txt를 만든다.

## 2. 확정된 결정사항

| 항목 | 결정 | 근거 |
|------|------|------|
| OCR 엔진 | Apple Vision (ocrmac 래퍼) 단독 | macOS 내장, 설치 최소(`pip install ocrmac`), 한글 인쇄물 인식률 상위. 사용자가 비교안(C안) 대신 A안 단독 선택 |
| 출력 형태 | 페이지별 txt + 합본 txt 둘 다 | 검수(페이지별)와 활용(합본) 모두 필요 |
| 작업 수준 | PoC: happy path만, 경로 하드코딩, 에러 처리/테스트/타입 힌트 생략 | CLAUDE.md PoC 규칙 |
| 언어 설정 | `language_preference=['ko-KR']` | 한국어 인식률 확보. 한자/혼합 레이아웃 문제 시 livetext 백엔드 시도 |

## 3. 구성

스크립트 1개: `ocr_book.py`

데이터 흐름:

1. 대상 폴더에서 `*.jpg` 수집
2. 자연 정렬: 파일명에서 숫자를 추출해 숫자 크기순 정렬
   (사전순 정렬은 "- 10.jpg"가 "- 2.jpg"보다 앞서므로 페이지 순서가 깨진다)
3. 각 이미지를 EXIF 회전 반영하여 열고 Apple Vision OCR 실행
4. 저장:
   - `output/pages/<원본파일명>.txt` — 이미지 1장당 1개
   - `output/book.txt` — `===== p.<숫자> =====` 구분선으로 이어 붙인 합본
5. 페이지마다 진행 로그 1줄 출력 (몇 번째/전체, 파일명)

## 4. PoC 검증 기준 (판정 1개)

Vision OCR은 처리 속도가 빨라 31장 전체를 한 번에 변환한 뒤,
그중 3장(곡면 왜곡이 있는 페이지 포함)을 골라 원본 이미지와 눈으로 대조한다.

**판정: 본문을 읽는 데 지장이 없는 수준인가?**

- 통과 → 결과 그대로 사용, 정식 구현 여부 결정
- 미흡 → PaddleOCR(오픈소스) 비교 투입을 재검토

## 5. 고려사항과 알려진 한계

- **곡면 왜곡**: 책 접합부 쪽 글자가 휘어 있어 해당 부분 오인식 가능. PoC에서는 측정만 하고, 심하면 정식 단계에서 이미지 전처리를 추가한다.
- **줄바꿈**: OCR 결과는 시각적 줄 단위라 문장 중간 줄바꿈이 남는다. PoC에서는 그대로 두고, 정식 단계에서 "문장부호로 끝나지 않으면 다음 줄과 병합" 규칙으로 문단을 복원한다.
- **꼬리말**: 페이지 하단의 "24 주식시장을 이긴 전략들" 같은 페이지 번호+제목이 본문에 섞인다. PoC에서는 그대로 둔다.
- **저작권**: 소장 도서의 개인 이용 목적 변환은 사적 복제 범위. 추출 텍스트의 외부 공유는 별개 문제.

## 6. 정식 구현 시 확장 항목 (PoC 범위 밖)

- 에러 처리(깨진 이미지, 빈 결과), 진행률 표시, 경로를 CLI 인자로
- 문단 복원 후처리, 꼬리말 제거 규칙
- 타입 힌트 100%, logging, 테스트
