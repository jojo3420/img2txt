판정: “baseline 기본 적용 유지”는 타당하지만, “전처리 투자 불필요”까지 종료하는 것은 과도합니다.

1. 부분 반대 — multiset은 보조 지표일 뿐입니다.

“진짜 놓침률”이 아니라 “정답 대비 글자 수 부족률”입니다.

- 같은 글자가 다른 위치에 있으면 오류를 숨깁니다.
- 불필요한 글자가 실제 누락을 상쇄할 수 있습니다.
- 읽기 순서, 필드 연결, 중복, 오인식을 평가하지 못합니다.
- 2010의 CER 0.181은 공백 포함, 놓침률 0.010은 공백 제거 후 계산되어 “94%가 어순 문제”라는 산식도 동일 조건 비교가 아닙니다.

따라서 A는 “현 CER을 절대 OCR 정확도로 쓰면 안 된다”까지만 동의합니다. 저장소 문서도 라벨 누락과 순서 문제를 이미 인정합니다([실험 문서](/Users/joel.silver/Workspace/gitroom/python/img2txt/docs/bench/2026-07-17-preprocess-ab-2010.md:50)).

2. 반대 — b1980으로 전처리 전체를 일반화할 수 없습니다.

- 세트별 앞 50페이지라 무작위 표본이 아닙니다([dataset.py](/Users/joel.silver/Workspace/gitroom/python/img2txt/img2txt/bench/dataset.py:43)).
- contrast는 2010만, upscale은 2010·b1980만 실행했습니다.
- 고정된 강도와 알고리즘 1개씩만 시험했습니다.
- 특히 b1980 deskew는 실제 회전이 2/50페이지뿐이었습니다.
- 0도 페이지도 JPEG로 다시 저장되어 재압축됩니다([preprocess.py](/Users/joel.silver/Workspace/gitroom/python/img2txt/img2txt/bench/preprocess.py:81), [저장 코드](/Users/joel.silver/Workspace/gitroom/python/img2txt/img2txt/bench/preprocess.py:107)). 즉 deskew 효과와 재압축 손상이 섞였습니다.

반대로 1980 deskew는 놓침률이 0.0573→0.0535로 줄었습니다. 작고 불완전한 증거지만 “어느 품질대에서도 개선 없음”은 반증됩니다.

3. 부분 동의 — 결정의 표현을 좁혀야 합니다.

유지 가능한 결정은 다음입니다.

> 현재 고정 설정 3종을 모든 문서에 기본 적용하지 않는다.

유지할 수 없는 결정은 다음입니다.

> 전처리 추가 투자 자체가 불필요하다.

실제 기울어진 문서만 선택 처리하거나, 완전한 정답으로 다시 측정했을 때 개선된다면 결정은 뒤집힐 수 있습니다. 특히 upscale의 11.4배 비용은 기각 근거가 강하지만, 조건부 deskew까지 닫을 근거는 부족합니다.

4. 반대 — 실험 종료 전 최소 재검증이 필요합니다.

- deskew 0도면 원본 파일을 그대로 사용해 재압축을 제거합니다.
- 실제 적용 각도와 적용 여부를 기록합니다.
- 결함 유형별로 뽑은 30~50페이지를 사람이 완전하게 교정합니다.
- 그 표본에서 위치 기반 평가와 페이지별 차이·불확실성 범위를 확인합니다.

전체 2,900페이지 재실험은 불필요합니다. 위 최소 검증 후 “기본 적용 기각”을 확정하면 됩니다.
