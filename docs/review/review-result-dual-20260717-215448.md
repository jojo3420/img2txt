## 3. 통합 권고사항 (Degraded, Codex 단일 + Claude 검증)

### 즉시 반영 (Must) — writing-plans 전 스펙 수정 권장
1. **델타 "상쇄" 과장 완화 (절 3-절 10).** "상수 편향이 델타에서 상쇄된다" → "동일 페이지-동일 참조의 paired 비교에서 편향 영향이 **부분 완화**된다. LLM이 텍스트를 재배열/추가하면 완화가 약해진다"로 하향. YAGNI(열검출 생략) 정당화도 이 완화된 논리에 맞춰 재서술.
2. **역할 재정의 일관성 (절 3, 상위 7.3).** 읽기순서 CER 델타를 "랭킹"이 아니라 "순서/문장성 **부작용 감시**"로. 랭킹은 상위 7.3 비용→속도 유지, 품질 하한=놓침률. 최종 모델 판정은 [놓침률 게이트 → 비용 → 속도] + [추가율-CER델타-degraded 부작용]의 결합 판정표로.
3. **빈 정답(N=0) 은폐 방지 (절 4).** 빈 정답+출력 있음을 별도 진단(예: `empty_ref_with_output` 플래그 또는 절대 초과 글자수)으로 기록. cer 관례(빈정답+출력=1.0)와 multiset(=0)의 불일치를 문서에 명시하고 정책 일치.

### 검토 후 반영 (Should)
4. **읽기순서 재정렬 실패 감지 진단 (절 5-절 8).** bbox_count-row_count-median_height-suspicious_layout_flag 등 메타를 리포트에 남겨 오묶음 이상치 페이지 추적 (열검출 고도화 없이 관측만).
5. **전처리 판정 규칙 구체화 (절 8).** "놓침률 개선 AND 추가율-degraded_page_count 악화 없음 → 채택 후보" 형태로 명시.
6. **테스트/좌표검증 강화 (절 5-절 9).** `_validate_bbox`가 0/음수 height, NaN, x/y 길이 불일치를 안전 처리(또는 ValueError). 다열/표-높이 상이 케이스 테스트 추가.
7. **놓침률 한계 강화 명시 (절 10).** 반복글자-필드이동-의미오류 미포착을 절 10에 추가.

### 참고 (Nice-to-have)
8. LLM 최종 랭킹은 지표 단독이 아니라 게이트+비교표(필드이동/형식보존 포함)로 — 스펙 7 판정 설계에서 처리(포인터).
9. micro 외 p50/p90 분위수 검토 (해석력).
10. multiset을 strict로 두는 근거 한 줄 추가 (절 4).
