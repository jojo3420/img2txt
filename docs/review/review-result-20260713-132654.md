## 2. Claude 크로스 리뷰

### 종합 소견
Codex 리뷰는 대부분 사실 기반이고 실제 공백을 정확히 짚었다. 특히 "채점 계약"(집계 공식/빈값/매칭 실패), "LLM 과잉 보정 부작용 지표", "허용 열화 탐색-평가 분리"는 설계 단계에서 반드시 확정해야 할 핵심이라 즉시 반영이 맞다. 일부(데이터 대표성, D8 문구, 비용 측정)는 스펙에 이미 부분 인지됐거나 실측 근거가 있어 부분 동의로 판정했다. 이번 스펙이 "설계 스펙까지"라도, 채점 방법론은 지금 못 박아야 다음 구현이 흔들리지 않는다.

### 동의하는 핵심 피드백 Top 3
1. 피드백 8 + 제안 4: LLM 부작용 지표. 평균 CER만 보면 일부 페이지를 심하게 망치는 과잉 보정을 놓친다. 가성비 판정의 신뢰성을 좌우.
2. 피드백 9 + 제안 3: 허용 열화의 탐색-평가 분리. 결과를 보고 합격선을 바꾸면 판정이 무의미. 채택 기준을 실험 전에 고정.
3. 피드백 3,4,5,6,16: 채점 계약(측정 단위 식별자/집계 공식/빈값/매칭 실패/리포트 스키마). 설계 단계에서 확정 안 하면 다음 구현이 흔들린다.

### 동의하지 않거나 조정한 피드백
- 피드백 11(비용 측정): codex exec가 토큰 총량("tokens used")을 실제로 출력함을 이번 실행으로 확인. "측정 불가"는 과장. 다만 입출력 분리/단가는 별도 확인이라 선행 게이트로 유지.
- 피드백 13(D8): 이미 7.4에 미검증을 명시했으므로 "충돌"보다는 D8 문구 정합성 문제. 문구 조정으로 해소.
- 제안 1(책 판정용 데이터 분리): 방향은 옳으나 현대 한글 책 공개셋이 없어 사용자가 공공문서를 택한 맥락이 있다. "책 소량 수작업 보조 정답"을 (1) OCR 판정 보조로 재고하는 선에서 절충 권고.

### Codex가 놓친 추가 관점
- corrector.py에 이미 길이 가드(LENGTH_GUARD_RATIO), 배치 실패 시 원문 유지, 개수 불일치 방어가 있다. 부작용 지표(8)는 이 기존 방어와 연계해 "가드 차단율"도 함께 리포트하면 저비용 모델의 불안정성을 조기 포착할 수 있다.
- 원시 OCR 지점 채점에서 줄 순서 문제: 우리 파이프라인은 좌표 기반 정렬(sort_lines_top_to_bottom)을 하므로, 정답 텍스트와의 순서 정합은 정규화의 공백 처리만으로 부족할 수 있다. 읽기 순서(reading order) 정렬 규칙을 채점 계약에 포함해야.

## 3. 최종 권고사항

### 즉시 반영 (Must) — 채점 방법론은 설계 단계에서 확정
- 채점 계약 확정: 측정 단위 식별자(page_id/segment_id + 문단 재결합/읽기 순서 규칙), CER/WER 집계 공식(micro 대 macro 확정), 빈 OCR = 정답 글자 전체를 누락 오류로 계산, 매칭 실패 = 기본 실험 중단(옵션 스킵).
- LLM 부작용 지표 추가: 악화 페이지 비율, 최악 악화폭, 삭제/추가량, 실행 실패율, 기존 가드 차단율.
- 허용 열화 탐색-평가 분리: 탐색셋에서 기준 확정 후 홀드아웃에 적용. 사후 조정 금지.
- 채택 우선순위 명문화(D8/7.3): 품질 하한 통과 -> 페이지당 비용 최저 -> 속도 동률 판정.
- 리포트 스키마 명시(16): run_id/데이터셋 해시/page_id/원문/정규화 출력/지표/시간/모델-프롬프트 버전/오류 상태.
- 부호 통일 매핑 표 + 엄격/관대 CER 이중 보고(7).
- D8 문구를 "가용성 확인 후 후보"로 조정(13).

### 검토 후 반영 (Should)
- 선행 가능성 게이트 신설: 모델 실호출 가능 여부, codex CLI 입출력 토큰/비용 노출 여부, AI Hub 자료 외부 LLM 전송 허용 여부를 구현 전 통과 조건으로.
- 전처리 설정값 고정(14): 대비 강도/확대 배율/deskew 각도 범위, 글자 잘림 실패 기록.
- 재현 메타데이터: macOS/ocrmac/codex CLI 버전, 프롬프트, 전처리 설정, 재시도/시간 제한.
- 데이터 대표성 보완: 책 소량 수작업 보조 정답을 (1) 판정 보조로 재고.

### 참고 (Nice to have)
- 판정용 규모 확대 계획(30~50p는 스모크 전용임을 명시).
